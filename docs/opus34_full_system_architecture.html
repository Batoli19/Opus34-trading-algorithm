Signal pipeline — 6 sequential gates
1
Kill zone check — bot_engine.py
Current UTC time checked against active sessions. London Open 06:00–09:00 UTC · London Close 15:00–17:00 UTC. Outside these windows: thread sleeps, no scanning occurs.
INSIDE → proceed to gate 2 OUTSIDE → sleep, skip entire cycle
↓
2
ICT signal detection — ict_strategy.py + ict_advanced_setups.py
M5 OHLCV bars pulled from MT5. Two setup detectors run in parallel:
CHOCH detector
Change of Character — structural market shift. Looks for broken swing high/low + confirmation bar. Returns: direction, entry zone, SL level.
LSR detector
Liquidity Sweep Reversal — stop hunt at key level then fade. Returns: direction, entry zone, SL level, sweep magnitude.
SIGNAL FOUND → proceed to gate 3 NO SIGNAL → end cycle, wait 10s
↓
3
Direction filter — ict_strategy.py (data-proven rules)
Hard-coded pair/direction rules derived from 16 months of backtest data. Zero overrides.
EUR/USD
BUY ✓ · SELL ✓ · Both profitable
GBP/USD SELL
SELL only ✓ · BUY bleeds −454p
USD/JPY BUY
BUY only ✓ · SELL bleeds −492p
ALIGNED → proceed to gate 4 WRONG DIRECTION → hard skip, log reason
↓
4
ML brain gate — brain_gate.py · XGBoost classifier
Signal features extracted and passed to trained model (04_BRAIN/models/entry_model.pkl). Model was trained on 1,123 trades with zero data leakage using time-series cross-validation.
28 input features
symbol · direction · setup type · session · hour of day · day of week · sweep magnitude · structural context + more
Output
Win probability 0.0 → 1.0. Threshold: 0.50
prob ≥ 0.50 → proceed to gate 5 prob < 0.50 → silently skipped, not logged as trade
Fail-safe: if model file missing → brain gate disabled, all signals pass through
↓
5
Risk manager check — risk_manager.py
Three sequential checks before any position opens:
Daily loss limit
−3.5% = $175 on $5k. Breached → bot stops for entire day
Loss streak pause
3 consecutive losses → 2hr forced pause. Resets on win.
Max open trades
3 simultaneous positions max. 4th signal → queue skip
ALL CLEAR → proceed to gate 6 LIMIT HIT → signal dropped, reason logged
↓
6
Position sizing — risk_manager.py
Lot size calculated from account balance, risk %, and SL distance in pips. Dynamic between standard and A+ setup.
Standard signal — 1.0% risk
$50 risk on $5k · lot = $50 ÷ (SL pips × pip value)
A+ signal — 1.5% risk
$75 risk on $5k · triggered by setup quality score
↓ signal passed all 6 gates

Activates when trade reaches +0.5R profit. Sets dynamic trailing threshold at 85% of peak. If price gives back >15% of peak unrealised profit → close immediately.
Profit lock

What happens inside a single 10-second cycle
t = 0ms · Cycle wakes
bot_engine.py scheduler fires · 3 pairs queued
t ≈ 5ms · Kill zone check
UTC time vs LO/LC windows. Exit if outside.
t ≈ 20ms · MT5 data pull
mt5.copy_rates_from_pos() · last 200 M5 bars per pair · pandas DataFrame
t ≈ 30ms · CHOCH + LSR scan
ict_advanced_setups.py runs detection logic on the 200 bars · returns signal or None
t ≈ 35ms · Direction filter
Hard lookup: GBPUSD + BUY → reject. Instant, no computation.
t ≈ 36ms · Brain gate
28 features assembled · xgboost.predict_proba() · <1ms inference · threshold 0.50
t ≈ 38ms · Risk check + sizing
Daily P&L pulled from trading_memory.db · lot size calculated · max positions checked
t ≈ 60ms · MT5 order send
mt5.order_send() · network round-trip to broker · confirmation received · position ticket stored
t ≈ 65ms · Log write
CSV row appended · SQLite updated · next pair starts its scan
t = 10,000ms · Next cycle
Scheduler wakes again · open positions also checked by trade_manager.py for giveback guard
